## 关键改进详解

### 1. 数据泄露修复

**原代码（危险）：**
```python
# 随机打乱时间顺序！
train_test_split(X, y, shuffle=True)
```

**修复后（安全）：**
```python
# 按时间顺序划分
tscv = TimeSeriesSplit(n_splits=5)
for train_idx, val_idx in tscv.split(X):
    # 训练集始终早于验证集
```

**影响：**
- 原版回测准确率可能虚高 10-20%
- 修复后回测结果更接近实盘表现

---

### 2. 涨停崩盘保护

**原问题：**
```
预测某日涨 9.8%
→ 用户挂单买入
→ 股票涨停，买不到
→ 或者涨停价买入，次日低开亏损
```

**修复方案：**
```python
if expected_return > 0.095:
    return BLOCKED, "接近涨停，可能无法成交"
```

---

### 3. 动态仓位管理

不再"全仓进出"，而是根据风险动态调整：

| 风险等级 | 基础仓位 | 实际仓位示例 |
|---------|---------|-------------|
| 安全 | 100% | 100% × 70% × 80% = 56% |
| 谨慎 | 60% | 60% × 60% × 70% = 25% |
| 警告 | 30% | 30% × 50% × 60% = 9% |
| 危险 | 0% | 0% |

（实际仓位 = 基础仓位 × 置信度 × 波动率因子）

## 实盘使用建议

### 第一阶段：模拟盘（1-2 个月）

1. 每天运行程序获取交易信号
2. 记录信号和实际结果
3. 观察风控是否正确触发
4. 统计：胜率、盈亏比、最大回撤

### 第二阶段：小资金实盘（1-2 个月）

1. 只投入总资金的 10%
2. 严格遵守风控建议
3. 每次交易不超过建议仓位
4. 严格执行止损

### 第三阶段：逐步加仓

1. 连续 2 个月盈利后，资金加到 30%
2. 持续监控风险指标
3. 每月回顾并优化参数

## 风险声明

⚠️ **重要提示：**

1. **这不是圣杯** - 任何模型都无法 100% 预测股价
2. **过去表现 ≠ 未来结果** - 回测盈利不代表实盘盈利
3. **风控不是万能的** - 极端行情下（如熔断、黑天鹅）所有模型都可能失效
4. **建议只用闲钱投资** - 不要借钱炒股，不要 all-in

**投资有风险，入市需谨慎。**
